# Splashdeals QA Onboarding Summary

## Current Testing Infrastructure

### E2E Testing (Playwright)
- **Directory**: `e2e/`
- **Main Suite**: `e2e/checkout.spec.ts`
- **Coverage**: Full payment lifecycle from checkout creation to webhook fulfillment.
- **Constraints**: 
  - Requires a running local dev server (`npm run dev`).
  - Requires Stripe CLI listening for webhooks (`stripe listen --forward-to http://localhost:3000/api/webhooks/stripe`).
  - Requires valid test IDs in `e2e/fixtures/test-data.ts`.
  - Most tests are currently `.skip` marked to prevent failures in headless/CI environments where the prerequisites are not met.
- **Commands**: 
  - `npm run test:e2e`: Runs all tests.
  - `npm run test:e2e:headed`: Runs tests in headed mode.

### Unit Testing (Vitest)
- **Status**: 🚧 Under Setup
- **Goal**: Cover Server Actions, utilities, and DTOs.
- **Architecture**: Will use `vitest` with `vite-tsconfig-paths` to resolve project aliases.

## Verification Protocol
The project uses Python-based verification scripts in `.agent/scripts/`:
- `verify_all.py`: Complete validation (Security, Lint, Schema, Tests, UX, SEO, Performance, E2E).
- `checklist.py`: Incremental validation.
- `test_runner.py`: Unified test execution that detects Node.js (vitest/jest) or Python (pytest) frameworks.
